# 6-7 分钟课堂讲稿

## 第 1 页：题目

各位老师同学好，我这次选择的是隐私保护方向，具体做机器逆学习。题目是“面向图像分类模型的机器逆学习：方法复现、隐私评估与改进”。这次课堂汇报讲研究背景、代码主线、评价指标和改进方法，最终报告里再补完整实验数值。

## 第 2 页：为什么选这个方向

金融模型会使用大量敏感数据，比如用户画像、交易行为、风险偏好和投资记录。现实中如果用户撤回授权，或者训练集中发现错误、违规数据，只删除数据库里的原始记录并不够，因为模型参数里仍可能保留这些数据的影响。所以我关注的问题是：怎样让一个已经训练好的模型删除指定数据的影响，同时尽量不损害其他数据上的能力。

## 第 3 页：课程资料给了什么

隐私保护作业里有两个方向：一个是端云协同联邦推荐，另一个是针对图像分类模型的机器逆学习。我选择第二个。当前仓库已经围绕 CIFAR100 Super20、ResNet18、Amnesiac、Bad Teacher、MIA/ZRF 指标和 Retain-Aware Bad Teacher 改进形成了完整主线，所以明天可以直接讲“问题、方法、指标、改进和实验计划”。

## 第 4 页：研究问题

这个任务可以抽象成三类数据和三个模型。forget set 是要删除影响的数据，retain set 是要保留能力的数据。Original model 是全量数据训练出来的模型，Gold model 是删掉 forget set 后完全重训的理想参照。我们希望用更低成本的逆学习方法，让学生模型尽量接近 Gold model。

## 第 5 页：现有方法

我会先复现两个基线。Amnesiac 方法的思路是抵消目标数据带来的训练影响。Bad Teacher 的思路是蒸馏：在保留数据上让学生模仿原模型，在遗忘数据上让学生模仿一个无效教师。它的优点是实现清晰，适合在课程代码里补全。

## 第 6 页：我的改进

我的改进叫 Retain-Aware Bad Teacher。普通 Bad Teacher 在 retain set 上主要模仿原教师的软标签，但纯蒸馏可能损伤保留类别的分类边界。我在 retain set 上额外加入监督交叉熵项，相当于给学生模型一个保留知识的锚点。总损失是双教师 KL 蒸馏损失，加上 λ 倍的 retain set 交叉熵损失。

## 第 7 页：实验怎么做

实验会比较五个对象：原模型、完全重训的 Gold model、Amnesiac、Bad Teacher、Retain-Aware Bad Teacher。指标不只看准确率，还会看 MIA 和 ZRF。MIA 衡量攻击者还能不能判断 forget 样本曾参与训练，ZRF 衡量模型在 forget set 上是否接近随机初始化教师。重点是判断两个问题：模型是否真的忘了指定数据，以及有没有破坏其他类别的性能。

## 第 8 页：预期贡献

最终我希望得到三个结果。第一，复现 Amnesiac 和 Bad Teacher 两条基线；第二，用 MIA 和 ZRF 补充准确率指标，避免只看表面准确率；第三，验证 Retain-Aware Bad Teacher 是否能改善遗忘彻底性和保留性能之间的折中。明天汇报到这里就足够，后续大报告再补实验表格和分析。
